大学

人总是选择性遗忘不开心的事情。
大学四年，生活上还是蛮快乐的，有这样那样的烦恼，但是又能算得了什么呢，即使宅了四年，还是觉得那是一段快乐的回忆。
么的浪漫的爱情，么的到处游玩，么的……大概我身边的人都是这样平凡吧，过了四年，自己成熟蛮多的，也认了命。
奥里给，干就完事了。

毕设

Idea

虽然我是一个瘦子，做饭大概是我为数不多的爱好。
学院那段时间分配老师之后，就开始定论文题目，当然我的指导老师尊重个人意见，我选择了自拟。前面说了我对做饭感兴趣，当然我毕设做的东西跟“吃”有关。
我观察到很多同学到饭点却不知道吃什么，你问我我问你，虽然吃的挺多的，但是这种情况真的很常见。我毕设的灵感就来于此，做一个“吃文化”的社区，具体内容呢，就是校园餐饮评价与美食推荐。

准备

当然论文题目选定了之后，就要开干了，既然做一个这样的社区应用，安卓肯定是首选了，后端的话用Spring Boot搭建。在这之前，我对Java和数据库已经比较熟悉，但是这样前后端分离的问题，我还没搞过，所以我得学OAuth2协议。还有安卓应用设计，之前大二的时候买了一本第一行代码，看了一点就没再看了，也就重新翻开这本书了。
2019年11月份就开始准备了，当时在学校附近一加做医疗系统的公司实习，这家公司不加班，6点多人就跑完了，还是蛮好的。回到宿舍我就开始对着第一行代码边看边敲。
12月份的时候，我打算月中就开工，当然这个想法落空了。在实现OAuth2中的密码模式时，我用了Spring Security OAuth2框架，中间看了一个视频，结果因为框架版本的问题和对基于代码配置的陌生，中间折腾了好久才搞定这个，这里要特别感谢航歌这个网站。
当然也遇到了另外一个问题，就是安卓应用设计，我的那本第一行代码是16年基于Android 7.0写的，2019年Android 10都来了，API变了好多，MVVM啥的，还有布局，泪崩。虽然把第一行代码中的代码敲了一半还多，结果到动手的时候蒙了，这里要感谢B站里longway777大大上传的视频，全部基于最新的api，一边看大佬视频，一边敲代码，连带看安卓开发者官网的文档，这样到了2020年2月初，终于开工了。

编码

2月初的时候，新冠肺炎疫情到了最严重的时候，学校回不去，我在家里也就老老实实实现我的社区应用了。
刚开始最困难的就是前端和移动端的框架搭建了，由于前面大量的准备，整体的框架也也大体上搭建完成了。后面就是设计数据库表结构和安卓UI了，当然中间也遇到了一些困难，一开始使用的Spring Data Jpa，后来由于这个玩意关联查询太恶心了，又无奈地去使用mybatis-plus。
还遇到一个困难就是安卓端https问题，安卓在4.0之后吧，就不再支持http了，我又去后端搞https，由于证书是自己生成的，安卓实际还是无法访问服务器数据，又经过了两天的折腾，这才搞定。
看着自己地应用越来越完善，还是非常开心的，后面又加了一些特色的功能。

论文

论文其实是两天赶出来的，4月20老师让交初稿，我3月末后就没写过应用了，后面一直在找工作。4月20的时候我跟说推迟一下，老师说4月30必须交。嗨，我到了4月28才开始写，两天赶完，人就是得逼。老师就提了一个格式上的问题，后来让室友看了一下，他提了大量的问题。/捂脸，确实太随意了，老师也没说啥。

答辩

由于我们专业人太多了，其实我的毕设指导老师并不是本专业老师，答辩的时候也在物联网专业组里。不过答辩老师有一个是我当时编译原理的老师，印象中，这个老师蛮好的，结果答辩那天刷新了我的认知。上来就怼，其实也就问了两个极其简单的问题，结果那哥们顾左右而言他，接着又怼，我都懵了，物联网的场子完全被他一个计科的老师占了。还好我在后面，到了下午5点才到我，老师们都急不可耐，ppt没演示完，让我演示，然后演示完，我把我github上的代码展示了一下，老师问了两个小问题结束。

总结

整个毕设，准备跟编码花的时间相当多，其实论文的撰写和答辩倒是没怎么费心思，有兴趣的话可以看下我的项目和论文。
